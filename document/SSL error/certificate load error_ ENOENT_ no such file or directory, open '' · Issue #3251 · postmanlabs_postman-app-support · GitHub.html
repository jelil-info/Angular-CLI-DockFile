harryi3t commented Nov 14, 2018

@adeniltonbs @dacloutier @Lavanya1689 @barbareek
The error that you see in the Postman Console is actually just a warning. Earlier Postman would not even send the request and error out if one of CRT file or Key file was missing. We fixed it by removing the restriction. Now, if only one file is used, we log the warning and continue sending the request.

If this is not working out for you as expected, could you help us out by providing some endpoints against which we can test this? A real-world API or even a sample server will work too.
